# Incident Report — INC-202

**Date:** 2026-02-03
**Severity:** P3
**System Affected:** Notification-Service
**Owner Team:** Data Engineering
**Incident Lead:** Chen Wei
**Referenced SOP:** SOP-05 — On-Call Escalation

## Summary

On 2026-02-03, a P3 incident was raised against **Notification-Service** due to
message queue backlog. The outage caused cascading failures in **Payment-Service** which depends
on Notification-Service for critical operations.

## Impact

- Notification-Service became unavailable for approximately 47 minutes.
- Downstream service **Payment-Service** experienced elevated error rates (>30%).
- Customer-facing operations managed by the **Data Engineering** team were degraded.

## Root Cause Analysis

The root cause was identified as: **message queue backlog**. Investigation conducted by
**Chen Wei** (Team Lead, Data Engineering) confirmed that Payment-Service has a hard
dependency on Notification-Service; when Notification-Service saturated its connection pool, all downstream
callers including Payment-Service began failing immediately.

## Resolution

Incident was resolved following **SOP-05** (On-Call Escalation).
Steps taken:
1. Paged on-call engineer via PagerDuty escalation per SOP-05.
2. Identified offending query pattern in Notification-Service logs.
3. Applied connection pool limit increase + circuit breaker configuration.
4. Verified Payment-Service recovery via health-check endpoints.

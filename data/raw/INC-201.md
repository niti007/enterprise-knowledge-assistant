# Incident Report — INC-201

**Date:** 2026-01-15
**Severity:** P2
**System Affected:** Auth-DB
**Owner Team:** Infrastructure
**Incident Lead:** Marcus Lee
**Referenced SOP:** SOP-01 — Incident Response Procedure

## Summary

On 2026-01-15, a P2 incident was raised against **Auth-DB** due to
connection pool exhaustion. The outage caused cascading failures in **UserProfile-API** which depends
on Auth-DB for critical operations.

## Impact

- Auth-DB became unavailable for approximately 47 minutes.
- Downstream service **UserProfile-API** experienced elevated error rates (>30%).
- Customer-facing operations managed by the **Infrastructure** team were degraded.

## Root Cause Analysis

The root cause was identified as: **connection pool exhaustion**. Investigation conducted by
**Marcus Lee** (Team Lead, Infrastructure) confirmed that UserProfile-API has a hard
dependency on Auth-DB; when Auth-DB saturated its connection pool, all downstream
callers including UserProfile-API began failing immediately.

## Resolution

Incident was resolved following **SOP-01** (Incident Response Procedure).
Steps taken:
1. Paged on-call engineer via PagerDuty escalation per SOP-05.
2. Identified offending query pattern in Auth-DB logs.
3. Applied connection pool limit increase + circuit breaker configuration.
4. Verified UserProfile-API recovery via health-check endpoints.

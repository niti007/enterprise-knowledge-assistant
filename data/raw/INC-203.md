# Incident Report — INC-203

**Date:** 2026-02-28
**Severity:** P2
**System Affected:** ReportingPortal
**Owner Team:** Data Engineering
**Incident Lead:** Chen Wei
**Referenced SOP:** SOP-02 — Data Backup and Recovery

## Summary

On 2026-02-28, a P2 incident was raised against **ReportingPortal** due to
DataWarehouse schema drift. The outage caused cascading failures in **DataWarehouse** which depends
on ReportingPortal for critical operations.

## Impact

- ReportingPortal became unavailable for approximately 47 minutes.
- Downstream service **DataWarehouse** experienced elevated error rates (>30%).
- Customer-facing operations managed by the **Data Engineering** team were degraded.

## Root Cause Analysis

The root cause was identified as: **DataWarehouse schema drift**. Investigation conducted by
**Chen Wei** (Team Lead, Data Engineering) confirmed that DataWarehouse has a hard
dependency on ReportingPortal; when ReportingPortal saturated its connection pool, all downstream
callers including DataWarehouse began failing immediately.

## Resolution

Incident was resolved following **SOP-02** (Data Backup and Recovery).
Steps taken:
1. Paged on-call engineer via PagerDuty escalation per SOP-05.
2. Identified offending query pattern in ReportingPortal logs.
3. Applied connection pool limit increase + circuit breaker configuration.
4. Verified DataWarehouse recovery via health-check endpoints.
